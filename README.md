# POSS-LOGIC-LM

This repository contains the experimental code, result files, generated figures, and source data used for the study of **Poss-Logic-LM**, a possibilistic beam-search extension of Logic-LM for robust symbolic reasoning with large language models.

The project compares several Logic-LM decoding and refinement configurations against the proposed Poss-Logic-LM approach on three logical reasoning benchmarks:

* **PrOntoQA**
* **ProofWriter**
* **FOLIO**

The experiments evaluate three open-weight language models:

* **Gemma-2-9B-Instruct**
* **Llama-3.1-8B-Instruct**
* **Qwen2.5-7B-Instruct**

## Main Code Components

### `src/logiclm_pipeline.py`

Implements the Logic-LM baseline pipeline. It uses a single symbolic formalization generated by the language model and evaluates it with the appropriate symbolic solver.

### `src/pbs_pipeline.py`

Implements the Poss-Logic-LM pipeline based on possibilistic beam search. It evaluates multiple candidate formalizations and aggregates the solver outputs using possibility and necessity degrees.

### `src/possibility.py`

Contains the core functions used to compute possibility degrees and necessity-based confidence scores.

### `src/self_refinement.py`

Implements the self-refinement mechanism used by Logic-LM variants when the solver returns parsing or execution errors.

### `src/metrics.py`

Contains the evaluation utilities used to compute accuracy, execution rate, and comparison metrics.

### `src/solvers/prolog_solver.py`

Wrapper for Prolog-based reasoning. It is used for PrOntoQA and ProofWriter.

### `src/solvers/prover9_solver.py`

Wrapper for Prover9-based theorem proving. It is used for FOLIO.

## Experimental Variants

The repository contains results for four Logic-LM variants and the proposed Poss-Logic-LM approach.

| Variant                | Description                                                   |
| ---------------------- | ------------------------------------------------------------- |
| `Logic-LM Greedy-noSR` | Greedy decoding without self-refinement                       |
| `Logic-LM Greedy-SR`   | Greedy decoding with self-refinement                          |
| `Logic-LM Beam-noSR`   | Beam decoding without self-refinement                         |
| `Logic-LM Beam-SR`     | Beam decoding with self-refinement                            |
| `Poss-Logic-LM`        | Possibilistic beam-search aggregation without self-refinement |

## Method Summary

Poss-Logic-LM extends Logic-LM by keeping multiple formalization candidates instead of relying only on the highest-ranked one. Each candidate is evaluated independently by the symbolic solver. The final prediction is obtained through possibilistic aggregation, and the necessity degree provides a confidence signal.

This allows the method to identify robust predictions when the beam agrees and fragile predictions when candidate formalizations disagree.
